# Speech Recognition Package

This folder contains ROS2 nodes and helpers for speech processing used in the project. It documents the responsibilities and interactions of the main modules and includes workflow diagrams for each component.

Modules:

- `asr` — Automatic Speech Recognition node: buffers audio, triggers transcription (VAD-driven), uses `faster_whisper` or local models and publishes `SpeechResult` messages.
- `vad` — Voice Activity Detection node: receives audio chunks, computes speech probability and publishes `Vad` messages.
- `wake_word` — Wake-word detector: sliding-window detector that publishes `WakeWord` messages when detected.
- `diarization` — Speaker diarization node: runs pyannote/diart pipeline in streaming mode, extracts embeddings and publishes `SpeechActivityDetection`.
- `database` — Speaker database manager: helpers to store and query speaker embeddings (optional, Mongo-backed in this repo).
- `ros_audio_source` — Audio source adapter: aligns incoming ROS audio chunks to block sizes expected by downstream components.

Where to find files

- Node implementations: `src/speech_recognition/speech_recognition/*.py`
- Workflows: `src/speech_recognition/*.mmd`

Quick notes

- Audio chunk size: audio is published by the audio capture node as fixed chunks (512 samples, ~0.032s @16kHz) on topic `/audio_and_device_info`.
- VAD drives activity detection and can throttle or gate downstream processing.
- Diarization may require HuggingFace authentication (`HF_TOKEN`) for gated models — set `HF_TOKEN` in environment or use `huggingface-cli login`.
- FFmpeg is required if you use tools that save or transcode audio to MP3.
